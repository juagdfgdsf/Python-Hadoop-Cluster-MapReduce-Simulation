Python Hadoop Cluster Simulation & MapReduce PipelineProject OverviewThis repository contains a ground-up Python simulation of an Apache Hadoop cluster [source: 1]. Designed to demonstrate the internal mechanics of Big Data frameworks without requiring physical server clusters or complex installations, this project simulates the Hadoop Distributed File System (HDFS), data fault tolerance, distributed MapReduce processing, and Artificial Intelligence/Machine Learning integration using purely standard Python libraries and local file system operations [source: 1].Key Features & ObjectivesHDFS Block Splitting: Simulates file ingestion by taking a tabular dataset (dataset.csv) and dividing it into fixed-size data chunks (5 rows per block) [source: 1].Distributed Storage Simulation: Routes data chunks in a round-robin pattern across three local worker directories representing physical Data Nodes (DataNode1, DataNode2, DataNode3) [source: 1].Master Controller (Name Node): Uses a centralized metadata JSON file (namenode.json) to track exact block locations, source nodes, and replication logs [source: 1].Fault Tolerance & Replication: Implements a $2\times$ replication factor [source: 1]. Every primary block gets a distinct replica generated and stored on an alternate Data Node, prepended with tracking metadata headers [source: 1].Custom MapReduce Engine: Features custom-built Map, Shuffle, and Reduce phases that parse distributed chunks to compute word frequencies, category distributions, and regional sales totals [source: 1].Native Machine Learning (KNN): Includes a K-Nearest Neighbors ($k=3$) classification algorithm built entirely from scratch (using custom Euclidean distance math and majority voting) to predict product categories based on encoded sales and region features [source: 1].System ArchitectureComponentReal-World Hadoop EquivalentSimulation ImplementationStorage EngineHDFSLocal file directories & .csv file blocks [source: 1]Master NodeName NodeCentralized namenode.json mapping file [source: 1]Worker NodesData Nodes3 separate local folders (DataNode1-3) [source: 1]ProcessingMapReduce FrameworkPython script pipelines (mapreduce.py) [source: 1]AI AnalyticsSpark MLlib / MahoutCustom-coded Python KNN model (ml_model.py) [source: 1]Execution Pipeline & Files1. main.py (HDFS & Replication Ingestion)Reads the raw 20-row dataset, generates 4 primary blocks, distributes them across the 3 Data Nodes, creates 4 distinct fault-tolerant replicas on alternate nodes, and outputs the cluster architecture map to namenode.json [source: 1].Bashpython main.py
2. mapreduce.py (Distributed Analytics)Reads active block paths directly from the Name Node metadata and executes three separate distributed analytics jobs [source: 1]:Task 1: Word Count across all product names [source: 1].Task 2: Product counts aggregated by Category [source: 1].Task 3: Total Sales revenue aggregated by Geographic Region [source: 1].Bashpython mapreduce.py
3. ml_model.py (AI Prediction Engine)Loads data via Name Node metadata, splits records into a 75/25 training/testing split (15 train, 5 test), and executes a custom K-Nearest Neighbors model [source: 1].Bashpython ml_model.py
